# 元图-知识图谱协同构建管理系统

## 产品概述

元图是一款面向企业级用户的知识管理与图谱构建平台。系统集成了大语言模型（LLM）技术，支持从非结构化文本和结构化数据中自动抽取知识实体与关系，并通过可视化方式呈现知识图谱，帮助用户高效构建、管理和应用领域知识图谱。

## 技术架构

### 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Vue 3 + Element Plus + ECharts |
| 后端框架 | Spring Boot 3.5.7 + MyBatis Plus |
| 关系数据库 | MySQL 8.1.0 |
| 图数据库 | Neo4j |
| 大模型集成 | 支持多种LLM API（如Qwen、GPT等） |

### 系统架构图

```
┌─────────────────────────────────────────────────────────────┐
│                        前端展示层                            │

├───────────────┬───────────────┬─────────────────────────────┤
│   MySQL       │    Neo4j      │        LLM API              │
│  关系数据存储  │   图数据存储   │     智能知识抽取             │

```

## 功能模块

### 一、系统仪表盘

系统首页提供全局数据概览，帮助用户快速了解系统运行状态：

- **数据统计卡片**：实时展示项目总数、实体类型数、关系类型数、语料数量、图谱节点数、图谱关系数
- **最近项目列表**：展示最近创建的项目及其节点、关系统计
- **最近语料列表**：展示最近添加的语料及处理状态
- **快速操作入口**：一键跳转至各功能模块

![仪表盘示意](./docs/dashboard.png)

---

### 二、项目管理

支持多项目隔离管理，每个项目拥有独立的图谱模型和数据空间：

**功能特性：**
- 创建、编辑、删除知识图谱项目
- 项目状态管理（启用/禁用）
- 项目描述与元信息维护
- 分页查询与搜索

**操作界面：**

| 字段 | 说明 |
|------|------|
| 项目名称 | 项目的唯一标识名称 |
| 描述 | 项目的详细说明 |
| 状态 | 启用/禁用状态控制 |
| 创建时间 | 项目创建的时间记录 |

---

### 三、图谱模型管理

为每个项目定义专属的图谱模型，包括实体类型和关系类型：

#### 3.1 实体类型管理

定义图谱中节点的类型，支持自定义颜色标识：

- 添加、编辑、删除实体类型
- 为每种实体类型设置显示颜色
- 实体类型描述信息维护

#### 3.2 关系类型管理

定义实体之间的连接关系：

- 添加、编辑、删除关系类型
- 关系类型语义描述
- 支持多种关系类型定义

**示例模型：**

```
实体类型：
├── 人物（颜色：#409EFF）
├── 组织（颜色：#67C23A）
├── 地点（颜色：#E6A23C）
└── 事件（颜色：#F56C6C）

关系类型：
├── 属于
├── 位于
├── 参与
└── 关联
```

---

### 四、知识图谱探索

核心功能模块，提供图谱可视化展示与智能知识抽取能力：

#### 4.1 图谱可视化

- **力导向图布局**：自动布局算法，清晰展示节点关系
- **交互操作**：支持缩放、拖拽、节点高亮
- **图例展示**：按实体类型分类显示
- **实时刷新**：数据更新后即时渲染

#### 4.2 LLM 智能知识抽取

集成大语言模型，实现从文本自动抽取知识：

**操作流程：**
1. 在文本框输入待抽取的文本内容
2. 点击"预览抽取结果"查看AI识别的实体和关系
3. 确认无误后点击"抽取并保存"将知识存入图谱

**抽取能力：**
- 自动识别文本中的实体（人物、组织、地点、时间等）
- 自动识别实体之间的关系
- 支持根据项目定义的模型进行抽取
- 抽取结果可视化预览

#### 4.3 图谱管理

- 刷新图谱数据
- 清空项目图谱（需二次确认）

---

### 五、语料管理

管理待处理的文本语料，为知识抽取提供数据来源：

**功能特性：**
- 添加、编辑、删除语料
- 语料来源标注
- 处理状态跟踪（未处理/已处理）
- 一键跳转知识抽取

**语料字段：**

| 字段 | 说明 |
|------|------|
| 标题 | 语料的标题 |
| 内容 | 语料的正文内容 |
| 来源 | 语料的来源渠道 |
| 状态 | 处理状态标记 |

---

### 六、结构化数据导入

支持从Excel/CSV文件批量导入结构化数据，快速构建图谱：

#### 6.1 导入流程

```
步骤1：上传文件
    ↓
步骤2：配置映射
    ├── 选择实体类型
    ├── 选择名称列
    └── 选择属性列
    ↓

    ├── 预览导入配置
    └── 执行导入操作
```

#### 6.2 支持的文件格式

- Excel 文件（.xlsx, .xls）
- CSV 文件

#### 6.3 导入特性

- 文件拖拽上传
- 数据预览功能
- 列映射配置
- 导入数量预估
- 导入完成后自动跳转图谱探索

## 系统特点

### 1. 智能化知识抽取

借助大语言模型技术，系统能够自动从非结构化文本中识别实体和关系，大幅降低人工标注成本：

- 支持多种LLM模型接入
- 可配置的抽取策略
- 抽取结果可预览、可修正

### 2. 多源数据融合

支持多种数据来源的知识构建：

- **非结构化文本**：通过LLM抽取
- **结构化数据**：Excel/CSV批量导入
- **手动录入**：直接在图谱中添加

### 3. 可视化图谱展示

采用ECharts图表库，提供专业的图谱可视化能力：

- 力导向布局算法
- 节点颜色分类
- 关系标签显示
- 交互式操作

### 4. 多项目隔离

支持创建多个独立的知识图谱项目，各项目拥有独立的：

- 图谱模型定义
- 语料数据
- 图谱数据

### 5. 企业级安全

- JWT身份认证
- 基于角色的权限控制
- 操作日志记录

## 应用场景

### 1. 企业知识管理

构建企业内部知识图谱，实现知识的结构化存储和智能检索。

### 2. 行业知识库建设

为特定行业（医疗、法律、金融等）构建领域知识图谱。

### 3. 智能问答系统

作为智能问答系统的知识底座，支持基于图谱的精准问答。

### 4. 关系分析与挖掘

分析实体之间的复杂关系，发现潜在关联。
